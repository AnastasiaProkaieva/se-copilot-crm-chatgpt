# Source Precedence and Freshness

Use the highest available source in this order:

1. Current approved security, compliance, legal, product, architecture, or pricing document
2. Current approved master RFP answer
3. Current product one-pager or battlecard
4. Recent opportunity-specific RFP response for customer-specific wording
5. Historical RFP response as a lead only, requiring verification

## Conflict handling

- Do not silently merge conflicting claims.
- Prefer the source with explicit approval status and the newest effective or review date.
- Present the conflict and route it to the named document owner or SME.

## Recommended metadata

For every reusable RFP response, capture:

- approval_status
- approved_by
- source_document
- source_section
- effective_date
- review_date
- product_version
- allowed_customer_scope
