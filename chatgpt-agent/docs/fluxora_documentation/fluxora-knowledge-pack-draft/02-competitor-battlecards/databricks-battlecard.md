# Internal-only status

This draft contains a **demo scenario**, discovery guidance, and evidence requirements. It does not contain a current, externally verified feature-by-feature comparison. Competitive Intelligence must approve public facts and Fluxora differentiators before use.

# Scenario observed in the supplied demo

The demo scenario records a customer data-science team with Databricks notebooks already in production; migration friction is the principal objection.

# What to discover

- Which Databricks workloads are in production: notebooks, jobs, SQL, ML, streaming, governance, or all of these?
- Which APIs, libraries, formats, catalogs, and operational workflows create switching cost?
- What business outcome is prompting evaluation despite the existing footprint?
- Which migration constraints are technical, organizational, contractual, or skills-related?
- Which success criteria would justify a phased migration or coexistence strategy?

# Positioning method

1. Acknowledge the customer's existing investment and avoid disparagement.
2. Anchor the evaluation in the customer's measurable workload, security, governance, migration, and commercial criteria.
3. Separate current Fluxora capabilities from roadmap items.
4. Use proof—benchmarks, architecture, migration plans, and references—rather than broad superiority claims.
5. Recommend coexistence or phased adoption when replacement is not justified.

# Proof package required before making a differentiation claim

- Migration assessment and compatibility matrix
- Phased coexistence architecture
- Validated performance and cost benchmark
- Reference architecture and customer evidence
- Operational runbook and skills plan

# Claims not to make

- Do not claim easier migration without a workload inventory.
- Do not claim lower TCO without a like-for-like model.
- Do not disparage notebooks or imply all workloads should be replaced.

# Battlecard gaps

- Current public competitor product scope and packaging
- Approved feature comparison by Fluxora product
- Verified win/loss evidence
- Approved objection responses
- Customer references
- Benchmark methodology
- Commercial comparison rules
- Last competitive review date

# Customer-safe response pattern

"Your current investment in Databricks is an important design constraint. We should first agree which workloads and outcomes are in scope, then validate Fluxora against an explicit benchmark and migration or coexistence plan."
