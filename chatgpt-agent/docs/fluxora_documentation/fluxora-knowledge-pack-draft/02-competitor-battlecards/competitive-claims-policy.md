# Core rules

- Distinguish **public competitor facts**, **customer-reported context**, **Fluxora positioning**, and **hypotheses**.
- Do not present customer-reported information as a universal competitor fact.
- Do not use absolute claims such as "faster," "cheaper," "more secure," or "easier" without an approved scope and evidence.
- Use workload-specific benchmarks with documented data, environment, configuration, concurrency, cache state, and cost assumptions.
- Do not quote confidential competitor pricing or non-public roadmap information.
- Do not imply legal, security, or compliance deficiencies without verified public evidence and Legal approval.
- Refresh public facts and battlecards on a defined cadence.

# Approved comparison structure

| Layer | Required content |
|---|---|
| Customer context | Existing footprint, goals, constraints, decision criteria |
| Competitor fact | Current public, cited information |
| Fluxora fact | Current approved product source |
| Difference | Narrow, evidence-backed comparison |
| Caveat | Scope, workload, version, region, or deployment dependency |
| Proof | Benchmark, architecture, migration plan, reference, or contract |

# Agent instruction

When the battlecard lacks a verified comparison, ask discovery questions and request the needed proof instead of inventing a differentiator.
