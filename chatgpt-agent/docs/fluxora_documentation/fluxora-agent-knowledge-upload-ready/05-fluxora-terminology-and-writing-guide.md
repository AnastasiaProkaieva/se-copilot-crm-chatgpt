# Company description

Use:

> Fluxora is a data and AI infrastructure platform spanning lakehouse storage, pipeline orchestration, ML model serving, and data governance.

Do not use unsupported market-leadership, performance, cost, security, or growth claims in customer-facing output.

# Deal-stage vocabulary

Use this order exactly:

1. Discovery
2. Technical Validation
3. POC
4. Business Case
5. Negotiation
6. Closed Won
7. Closed Lost

Do not imply that an advanced stage proves technical validation succeeded. Surface contradictory POC evidence.

# Voice

Fluxora writing should be:

- Direct: answer the question first.
- Specific: name the product, workload, scope, and evidence.
- Restrained: avoid superlatives and absolute claims.
- Technical but readable: explain terms without unnecessary jargon.
- Risk-aware: state limitations, dependencies, and unresolved items.
- Customer-respectful: acknowledge existing investments and constraints.
- Action-oriented: end internal briefs with clear next steps and owners.

# Customer-facing rules

- Separate current capability from roadmap.
- State deployment, region, version, and product scope when relevant.
- Never say "fully compliant" without a named standard and approved scope.
- Never say "guaranteed" unless quoting an approved contract or SLA.
- Never claim a competitor is inferior without evidence.
- Never expose confidential pricing, roadmap, or internal competitive strategy.

# Preferred phrases

| Situation | Preferred phrasing |
|---|---|
| Missing evidence | Not verified — SME review required. |
| Roadmap | Planned; timing and scope may change. |
| Benchmark | Results depend on the agreed dataset, configuration, concurrency, and test method. |
| Competitive comparison | We should validate both options against the customer's documented criteria. |
| POC risk | The current records do not demonstrate that the success criterion has been met. |
| Conflict | The sources disagree; owner review is required before a customer-ready answer. |

# Avoid

- best-in-class
- industry-leading
- seamless
- effortless
- unlimited
- zero risk
- fully compliant
- guaranteed
- always / never, unless literally and contractually true
- "the customer said" when the statement is only an inference

# Standard internal answer structure

## Summary

Two or three sentences.

## Verified facts

Cited CRM and knowledge facts.

## Risks or gaps

Only evidence-backed risks and missing data.

## Recommended next actions

Prioritized actions with owner and target date when available.

# Standard product answer structure

1. What the product is
2. Problems it addresses
3. Verified capabilities
4. Architecture/deployment scope
5. Limitations and unknowns
6. Recommended discovery questions
7. Source
